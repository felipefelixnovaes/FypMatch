## 1. RESEARCH PROTOCOL — ICP FYPMATCH BR

### ICP A — LGBTQIA+ Amplo (Segmento Primário)

**Perfil demográfico:**
- Idade: 20-30 anos
- Gênero: todos (ênfase em mulheres bi/pan, homens bi, não-binários, trans)
- Localização: SP, RJ, BH, Curitiba, Porto Alegre, Recife
- Renda: classes B/C (R$2.500–8.000/mês)

**Comportamento digital:**
- Ativos no Instagram (stories, Reels), TikTok e Twitter/X
- Seguem: perfis de diversidade, psicologia, relacionamentos, saúde mental
- Apps instalados: Tinder ou Bumble (usuários há 6+ meses com frustração)
- Engajamento alto com conteúdo de: "relacionamento saudável", "comunicação não violenta", "identidade LGBTQIA+"

**Dores reais:**
- Grindr é "só para homens gays e foco em sexo"
- Tinder é heteronormativo no algoritmo e nos usuários
- Bumble não resolve a inclusão bi/pan/trans
- Cansaço de ser fetichizado ou não reconhecido

**Gancho de conversão:**
> "Um app que realmente te inclui — todos os gêneros, todas as orientações, sem fetichização."

---

### ICP B — Neurodiverso (Segmento Secundário / Alta Defensabilidade)

**Perfil:**
- Idade: 20-35 anos
- Diagnóstico: TDAH, TEA (Transtorno do Espectro Autista), ansiedade social
- Contexto: universitários, profissionais júnior, pessoas em processo de autodiagnóstico

**Comportamento digital:**
- TikTok heavy (comunidade #TDAH, #Autismo no TikTok BR é enorme)
- Reddit (r/TDAH_brasil, r/autismo)
- YouTube: vídeos sobre neurodiversidade, saúde mental
- Resistência a publicidade direta — preferem conteúdo orgânico/UGC

**Dores reais:**
- Apps de relacionamento são socialmente exaustivos
- Não saber o que dizer após um match
- Medo de parecer "estranho" ou "intenso"
- Precisam de mais contexto para interpretar mensagens

**Gancho de conversão:**
> "Um conselheiro IA que te ajuda a navegar relacionamentos do jeito que faz sentido para você."

---

### ICP C — Frustrado com Superficialidade (Porta de Entrada em Escala)

**Perfil:**
- Idade: 23-32 anos
- Qualquer gênero/orientação
- Usou Tinder por 6+ meses, matches que nunca viraram conversa

**Dores reais:**
- "Fico horas swipando e nada acontece"
- "Matches que somem depois de 2 mensagens"
- "Parece que todo mundo quer só validação, não conexão real"

**Gancho de conversão:**
> "Chega de swipe infinito. Matches com score de compatibilidade real."

## 6. MAPEAMENTO FIREBASE ANALYTICS — FUNIL COMPLETO

```javascript
// Eventos a implementar em ordem de funil

// 1. Instalou o app
analytics.logEvent("app_open", { source: "install" })

// 2. Completou cadastro
analytics.logEvent("sign_up", {
  method: "google",
  platform: "android" // ou "ios"
})

// 3. Perfil completo (foto + bio + preferências)
analytics.logEvent("profile_complete", {
  has_photo: true,
  has_bio: true,
  orientation_set: true
})

// 4. Primeiro swipe
analytics.logEvent("first_swipe", {
  direction: "right" // ou "left"
})

// 5. Primeiro match
analytics.logEvent("first_match", {
  days_since_install: N,
  compatibility_score: N
})

// 6. Primeira mensagem enviada
analytics.logEvent("first_message", {
  hours_since_match: N,
  used_ai_suggestion: true // ou false
})

// 7. Usou conselheiro IA
analytics.logEvent("ai_credit_used", {
  credits_remaining: N,
  session_type: "icebreaker" // ou "anxiety", "compatibility", etc
})

// 8. Viu paywall
analytics.logEvent("paywall_viewed", {
  trigger: "who_liked_me" // ou "credits_empty", "daily_limit", "d7_offer"
})

// 9. Assinou Premium
analytics.logEvent("subscribe", {
  plan: "premium", // ou "vip"
  price: 19.90,
  currency: "BRL",
  trigger: "who_liked_me" // trigger que causou a conversão
})

// 10. Upgrade para VIP
analytics.logEvent("vip_upgrade", {
  previous_plan: "premium",
  days_as_premium: N
})
```

## 7. KILL / SCALE RULES

### Kill Rules (pausar ad set / criativo)

| Condição | Ação | Janela de avaliação |
|----------|------|-------------------|
| CPI > R$6,00 | Pausar ad set | 3 dias com ≥ R$30 gasto |
| CTR < 0,8% | Pausar criativo | 2 dias com ≥ 1.000 impressões |
| CPC > R$2,50 | Pausar criativo | 2 dias |
| 0 installs com R$20 gasto | Pausar ad set | Imediato |
| Frequência > 3 em 7 dias | Rodar novos criativos | Semanal |

### Scale Rules (aumentar orçamento)

| Condição | Ação | Incremento |
|----------|------|-----------|
| CPI < R$3,00 por 3 dias | +20% orçamento | Máx 1x/semana |
| CPI < R$2,00 por 5 dias | +50% orçamento | Máx 1x/semana |
| ROAS > 2x por 7 dias | Criar lookalike 1% | Nova audiência |
| CTR > 3% | Testar em audiências maiores | Duplicar ad set |
| Sign_up rate > 40% | Escalar campanha | CBO com evento sign_up |

### Fase Learning (primeiros 7 dias por ad set)
- NÃO fazer alterações durante fase de aprendizado
- Budget mínimo: R$30/dia por ad set para sair da aprendizagem
- Evento de otimização inicial: **install** (mais volume) → depois trocar para **sign_up**
